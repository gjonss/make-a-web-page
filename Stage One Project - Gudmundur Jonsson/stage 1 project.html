Key Ideas

Section 1: The Web

The World Wide Web

The web consists of c. 30 billion pages today and was invented in the 1990s. It largely consists of pages that are generated by HTML code (HyperText Markup Language), and pages link to each other through Hyper Links.

The Major Components of the Web

The web can largely be broken down into 3 major component parts; 1) The user, computer and browser (e.g. Chrome, Internet Explorer, Firefox, Safari) 2) The internet, being a collection of hyperlinked webpages largely constructed using HTML 3) Servers that contain and store large repositories of data.

A user accesses the internet through a web browser and information is relayed to and from servers via a protocol; HTTP. This protocol determines how information is relayed across the internet.

Section 2: Introduction to HTML

HTML Basics

HTML is made up of; 1) text content, 2) markup, 3) references to other documents (e.g. videos, images), 4) links to other pages.

Markup determines how text will ultimately look on a webpage, and it comes in the form of tags. Tags generally consist of three parts; 1) opening tag, 2) content, 3) closing tag. These three parts are together referred to as an 'element'.

Examples of commonly used HTML tags are:

  • 1) <b>content</b>
  • This tag makes the text bold.
  • 2) <em>content</em>
  • This tag makes the text italicised.

Note that if a closing tag is required to determine how much content the tags apply to. If it is missing, all of the content following on from the opening tag will be e.g. bold or italicised.

HTML Attributes

HTML tags can have attributes that determine the behaviour of the tag. They usually appear in the following form:

  • <TAG ATTR="value">content</TAG>

An example of a tag that uses attributes is the anchor tag: <a>. The anchor tag applies a hyperlink to the text:

  • <a href="https://www.reddit.com">derp</a>

In this case, the 'derp' text will appear in blue and underlined, indicating that the text is hyperlinked, i.e. by clicking on the text, the user will be taken to www.reddit.com.

Another tag that uses attributes is the image tag. It comes in the following form:

  • <img src="ulr" alt="text">

The 'src' attribute indicates the source of the image, i.e. website address, and the 'alt' attribute indicates the text that should appear in case the image cannot be accessed.

Whitespace

All spaces in text, whitespaces, are largely ignored in HTML. E.g. consider the following text:

  • This is the        text content
    in a      sentence

This text would appear as a single sentence without a linebreak and with only one space between each word.

In order to enter a linkebreak, the <br> tag is used (break). This is a so called void tag, i.e. is does not require a closing tag. The <p> tag can also be used, i.e. the paragraph tag, although this tag requires a closing tag.

Inline vs. Block Elements

Tags can be broken down into two types;

  • 1) inline - used within a sentence, e.g. <br> <b> <em> <img> <span>,
  • 2) block - frames the text within an invisible box, e.g. <p> <div>.

Inline elements do not cause text to wrap down into the next line while block elements do.

Span/Div Elements

Both of these elements are so called 'container' elements with <span> being inline and <div> being block. Both of these elements have a 'class' attribute where CSS styles and Javascript code can be used to alter the visual style of the content:

  • <span class="foo">text</span>
  • <div class="bar">text</div>

This links to a CSS file (in the same directory as your HTML file) that outlines the styles used in your HTML file. In the above example, the CSS file is called 'style.css'.

HTML Layout

All HTML documents require several component parts in order to fulfil the requirements of HTML markup. They are as follows:

  • <!DOCTYPE HTML>
  • <html>
  • <head>
  • <title>Title!</title>
  • </head>
  • <body>
  • <b>content</b>
  • </body>
  • </html>

The first tag, <!DOCTYPE HTML>, indicates the type of page it is. The opening and closing <html></html> tags surround the entire html document. The <head></head> tags contain metadata, CSS/Javascript styling information, and the <title> tag, indicating the overall title of the document. The<body> tags contain the contents of the document.

Section 3: Webpages, Documents and Structure

HTML, CSS and Javascript

HTML is the structure of the website, and CSS components represent the style of your site. Javascript components are interactive elements of the website.

Developer Tools

Google Chrome includes 'developer tools'. They allow you to see the structure of the page, i.e. the different HTML elements that the page is made up of. These tools also allow you to see the CSS styles that apply to the page. CSS stands for ‘Cascading Style Sheets’.

HTML Structure

HTML code is represented in a tree-like structure, with elements contained within other parent elements.

The 'Document Object Model' (DOM) represents the tree-like structure of a page.

CSS allows you to indicate fonts, font sizes, font colour, borders etc.

Every webpage consists of boxes, with the page broken down into a series of boxes, even boxes within boxes.

You can right-click on any part of a webpage and choose 'inspect element'. This will bring up the developer tools, allowing you to see the HTML code and associated CSS style information. Here you can change the style and see the changes immediately appearing on the webpage. To get back to the original page design, simply refresh the page.

Cascading Style Sheets (CSS)

You can link your HTML file to a CSS stylesheet file by using the <link> tag in your <head> element. This is a void tag (requires no closing tag).

This could come in the form of:

  • <link rel="stylesheet" type="text/css" href="style.css">

This links to a CSS file (in the same directory as your HTML file) that outlines the styles used in your HTML file. In the above example, the CSS file is called 'style.css'.

You can right-click on any part of a webpage and choose 'inspect element'. This will bring up the developer tools, allowing you to see the HTML code and associated CSS style information. Here you can change the style and see the changes immediately appearing on the webpage. To get back to the original page design, simply refresh the page.

An example of a style in the CSS file could be:

  • .description {
           color: red;
    }

This would apply the colour red to all elements that use the class 'description'.

Alternatively, if you do not want to use a separate file to indicate styling for your HTML file, you can use the <style> tag within your HTML file:

  • <style>
    .description {
           color: red;
    }
    </style>

In this example, the style tag includes CSS code that you would otherwise use in your separate CSS file. You can also include styling as an inline element of your HTML code, although this is not recommended. Inline styling properties become particularly time-consuming to change for multiple webpages.

Inheritance in CSS

Inheritance is a feature in CSS that relies upon the ancestor-descendant relationship. This idea means that properties are not applied only to a specific element but also to its nested descendants within the document tree. Descendant elements may inherit CSS property values from enclosing ancestor elements. Most text related properties are inherited, while box related properites are generally not. Inheritance prevents code from being declared repeatedly within a style sheet. This means less CSS has to be written. It also means that webpages will load faster.

Please read this inheritance section on the CSS Wikipedia page for further information.

Example of Inheritance

Consider the following consecutive snippets of CSS and HTML code:

  • h1 { color: pink;}
  • <h1>This is to<em>illustrate</em>inheritance</h1>

The h1 element contains an emphasizing element. The word 'illustrate' will inherit the pink color of the h1 parent element due to the h1 styling information in the CSS code.

CSS Code

CSS code controls the 'style' of HTML elements. An example of CSS code that makes all h1 elements have a blue background and yellow text would be:

  • h1 { background-color: blue; color: yellow; }

This tells the browser that this styling information should be applied to all h1 elements on the page. In this case, h1 is a selector.

All of the information between the curly braces {} is interpreted as a rule that is applied to every h1 element.

The code 'background-color: blue' is a declaration, with 'background-color' being a property and 'blue' being a value associated with that property.

CSS Class Selectors

The above example shows how we set styling information for all h1 elements that appear in the HTML code. If we only want to add styling to some h1 elements, we can use CSS class selectors instead.

Consider the following HTML code (also contains the CSS styling information within the <style> element):

  • <h1 class="green">Green</h1>
  • <h1 class="pink">Pink</h1>
  • <h1 class="red">Red</h1>
  • <style>
  • h1 {
  •        text-align: center;
  •        background-color: blue;
  • }
  • .green {
  •        color: green;
  • }
  • .pink {
  •        color: pink;
  • }
  • .red {
  •        color: red;
  • }
  • </style>

This example shows how we can apply rules to all h1 elements (i.e. they are all centered and with a blue background), or only some (e.g. each h1 element is in a different font color). You can see that each h1 element includes a class selector (e.g. 'green') that is then defined as part of the CSS coding.

CSS Structure and Avoiding Repetition

The use of CSS styling as a separate CSS file avoids the need to declare styling information repeatedly within the HTML code. This saves time, particulary in cases where the CSS information is associated with multiple HTML files. It also reduces the likelihood of error as you only have to declare the styling information once within the CSS file. Be aware, however, that CSS styling is applied according to the specificity it carries. This indicates the relative weights of various styling rules, and determines which styles are applied in cases where more than one rule applies.

Please see this Wikipedia entry on specificity for further information.

Webpages and Boxes

All webpages can be broken down into rectangular spaces. A front-end web developer begins by taking a page design and breaking it down into such spaces (boxes), with smaller boxes possibly residing within larger boxes etc. In this way, a seemingly complex layout can be broken up into multiple rectangular elements. This is where it is particularly useful to break code up using the div element. Associated div class selectors can also use Flexbox layout modes to align divs e.g. next to each other, although note that this can only work for some web browsers that use the latest version of the CSS standard.

Please see this layout website for further information on the use of Flexbox and much more besides!
